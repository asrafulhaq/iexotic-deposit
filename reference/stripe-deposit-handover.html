Section 01

📋 Project Overview

The client rents vehicles and needs to place a refundable security deposit hold on customers' cards. Stripe's standard payment links don't support this — they charge immediately. This project builds a custom one-button webpage that creates a Stripe Checkout Session with capture_method: manual, which holds funds without collecting them.

Customer experience

Identical to a normal Stripe payment link. They land on a page, click one button, enter card details on Stripe's hosted page, and see a confirmation. No confusion.

Owner experience

Gets a Slack notification instantly when a card is authorized. After the car is returned, they log into the Stripe Dashboard and either capture (charge) or release (cancel) the hold.

Key technical distinction

The capture_method: manual flag must be set inside payment_intent_data, NOT at the top level of the Checkout Session. Stripe's default authorization window is 7 days. If holds longer than 7 days are needed, Extended Authorization must be enabled in the Stripe Dashboard (allows up to 30 days).

api/create-checkout.js

The core serverless function. The deposit amount is read from the DEPOSIT_AMOUNT_CENTS environment variable so it can be changed without touching code. Falls back to $500 if the variable is missing.

JS api/create-checkout.js Creates Stripe checkout session 
const Stripe = require('stripe');
const stripe = Stripe(process.env.STRIPE_SECRET_KEY);

module.exports = async (req, res) => {
  // Only accept POST requests
  if (req.method !== 'POST') {
    return res.status(405).json({ error: 'Method not allowed' });
  }

  try {
    // Amount from env variable (in cents). Change DEPOSIT_AMOUNT_CENTS in Vercel
    // to update the deposit without touching code.
    // Examples: 50000 = $500 | 30000 = $300 | 100000 = $1,000
    const amount = parseInt(process.env.DEPOSIT_AMOUNT_CENTS, 10) || 50000;

    const session = await stripe.checkout.sessions.create({
      mode: 'payment',
      payment_method_types: ['card'],

      payment_intent_data: {
        // ⚠️  This is the critical flag — hold funds, do NOT charge yet
        capture_method: 'manual',
        description: 'Vehicle Rental Security Deposit',
      },

      line_items: [{
        price_data: {
          currency: 'usd',
          product_data: {
            name: 'Security Deposit — Vehicle Rental',
            description: 'Refundable hold. Released after vehicle return and inspection.',
          },
          unit_amount: amount,
        },
        quantity: 1,
      }],

      success_url: `${process.env.SITE_URL}/success.html`,
      cancel_url:  `${process.env.SITE_URL}/`,
    });

    // Redirect customer to Stripe's hosted checkout page
    res.redirect(303, session.url);

  } catch (err) {
    console.error('Stripe error:', err);
    res.status(500).json({ error: err.message });
  }
};

api/webhook.js

Receives events from Stripe after a card is authorized. Verifies the signature to confirm it's genuinely from Stripe, then sends a Slack message. Body parsing must be disabled so the raw request body is available for signature verification.

JS api/webhook.js Stripe webhook → Slack alert 
const Stripe = require('stripe');
const stripe = Stripe(process.env.STRIPE_SECRET_KEY);

// Disable Vercel's automatic body parsing.
// Required so we can verify Stripe's signature on the raw body.
module.exports.config = {
  api: { bodyParser: false },
};

module.exports = async (req, res) => {
  if (req.method !== 'POST') {
    return res.status(405).send('Method not allowed');
  }

  // Collect the raw request body as a Buffer
  const rawBody = await new Promise((resolve, reject) => {
    const chunks = [];
    req.on('data', chunk => chunks.push(chunk));
    req.on('end',  ()    => resolve(Buffer.concat(chunks)));
    req.on('error', reject);
  });

  // Verify the event actually came from Stripe
  let event;
  try {
    event = stripe.webhooks.constructEvent(
      rawBody,
      req.headers['stripe-signature'],
      process.env.STRIPE_WEBHOOK_SECRET   // whsec_... from Stripe Dashboard
    );
  } catch (err) {
    console.error('Webhook signature failed:', err.message);
    return res.status(400).send(`Webhook Error: ${err.message}`);
  }

  // This event fires when customer completes checkout (hold is active)
  if (event.type === 'checkout.session.completed') {
    const session = event.data.object;

    const amount = `$${(session.amount_total / 100).toFixed(2)}`;
    const name   = session.customer_details?.name  || 'Unknown';
    const email  = session.customer_details?.email || 'Unknown';
    const piId   = session.payment_intent;

    const slackPayload = {
      text: [
        '🔒 *Security Deposit Authorized*',
        `*Amount held:* ${amount}`,
        `*Customer:* ${name}`,
        `*Email:* ${email}`,
        `*Payment Intent ID:* \`${piId}\``,
        '',
        '→ <https://dashboard.stripe.com/payments|Open Stripe Dashboard> to capture or release.',
      ].join('\n'),
    };

    try {
      await fetch(process.env.SLACK_WEBHOOK_URL, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(slackPayload),
      });
    } catch (slackErr) {
      // Don't fail the webhook if Slack is down — Stripe needs a 200
      console.error('Slack notification failed:', slackErr);
    }
  }

  // Always return 200 to Stripe or it will retry the webhook
  res.status(200).json({ received: true });
};

Section 08

💬 Slack Webhook Setup

The client does this in their own Slack account. It takes about 2 minutes and is free.

  1. Go to api.slack.com/apps

    Sign in with the client's Slack account. Click Create New App → choose From scratch.

  2. Name and pick workspace

    Name it anything — e.g. "Deposit Alerts". Select the client's Slack workspace. Click Create App.

  3. Enable Incoming Webhooks

    In the left sidebar under Features, click Incoming Webhooks. Toggle the switch to On.

  4. Add webhook to workspace

    Scroll down and click Add New Webhook to Workspace. Pick the channel where alerts should appear (e.g. #deposits or #rentals). Click Allow.

  5. Copy the URL

    A URL starting with https://hooks.slack.com/services/... will appear. Copy it. This is the SLACK_WEBHOOK_URL environment variable.

Section 09

⚡ Stripe Webhook Setup

This registers your deployed URL with Stripe so it knows where to send events.

Do this after the first deployment

You need your live Vercel URL first. Complete the initial deploy (Section 10, steps 1–6), then come back here.

  1. Open Stripe Dashboard → Developers → Webhooks

    Use test mode during development, switch to live mode for production.

  2. Click "Add endpoint"

    Endpoint URL: https://YOUR-VERCEL-URL.vercel.app/api/webhook

  3. Select the event to listen for

    Search for and select checkout.session.completed. This fires when a customer completes the Stripe checkout form (hold is placed).

  4. Click "Add endpoint" to save

    Stripe will show the endpoint details page.

  5. Copy the Signing Secret

    Click Reveal under "Signing secret". Copy the whsec_... value. This is your STRIPE_WEBHOOK_SECRET environment variable. Add it to Vercel settings.

  6. Redeploy after adding the secret

    Run vercel --prod again or trigger a redeploy in the Vercel dashboard so the new env var takes effect.

Section 10

🚀 Deployment Steps

  1. Create the project folder

    Make a folder called deposit-link. Create all 6 files inside it exactly as shown in Section 06.

  2. Install dependencies locally (for testing)

    Run npm install in the project folder. This creates node_modules/ locally.

  3. Deploy to Vercel

    Run vercel --prod in the project folder. On first run it asks for project name and settings — accept all defaults. Note the URL Vercel assigns (e.g. deposit-link-xxx.vercel.app).

  4. Add environment variables in Vercel Dashboard

    Go to vercel.com → your project → Settings → Environment Variables. Add all 5 variables from Section 07. For the first deploy, use sk_test_ Stripe key.

  5. Set up Stripe webhook (Section 09)

    Now that you have the Vercel URL, register the webhook endpoint in Stripe Dashboard and copy the signing secret back to Vercel.

  6. Redeploy to pick up env vars

    Run vercel --prod again, or click Redeploy in the Vercel dashboard. Env vars only take effect after a deploy.

  7. Test with Stripe test card (see Section 11)

    Open the live URL, click the button, complete the Stripe checkout with the test card. Verify the hold appears in Stripe Dashboard and the Slack alert fires.

  8. Go live: swap to live Stripe keys

    In Vercel env vars, replace sk_test_ with sk_live_. Also update STRIPE_WEBHOOK_SECRET — it changes between test and live mode. Redeploy. Send the URL to the client.

Section 12

📖 Owner's Workflow (Post-Deploy)

Once live, the client manages everything from Stripe Dashboard → Payments. No technical knowledge needed for day-to-day operation.

When a customer authorizes a deposit

A Slack alert fires instantly. The payment appears in the Stripe Dashboard with status Uncaptured. No money has moved yet.

After the vehicle is returned

In Stripe Dashboard → Payments → click the payment → two options appear:

  • Capture — Charges the customer. Use this if there is damage or outstanding balance.
  • Cancel payment intent — Releases the hold with no charge. Use this after a clean return.

Changing the deposit amount

Go to Vercel Dashboard → Project → Settings → Environment Variables → update DEPOSIT_AMOUNT_CENTS → trigger a redeploy. Also update the display amount in index.html and redeploy. The new amount takes effect immediately for all future sessions. Existing authorized holds are unaffected.

Authorization window — 7 days

Stripe's default hold expires after 7 days. If the client needs longer (e.g. week-long rentals with buffer time), they must enable Extended Authorization in Stripe Dashboard → Settings → Extended Authorization (allows up to 30 days). Flag this to the client during handover.
